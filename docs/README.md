# docs

Project documentation. The Master Spec is the single source of truth.

## Layout

| Path | Contents |
|---|---|
| `oracle/` | The Master Spec Oracle (`v1.2`) — the canonical product/tech specification. Place the docx here. |
| `planning/` | The Feature Backlog and any phase-specific planning artifacts. |
| `qa/` | QA test runs, accessibility audits, regression notes. |

When you copy in the spec and backlog docs from the project root, place them at:

- `docs/oracle/Zazenware_MVP1_Master_Spec_v1_2.docx`
- `docs/planning/Zazenware_MVP1_Feature_Backlog_v1_0.docx`
