Our commitment

zazenware aims to meet the Web Content Accessibility Guidelines (WCAG) 2.1, Level AA, across every page of the site. Accessibility is a design value, not a checkbox — we want anyone who wants to browse our art or place an order to be able to do so.

We treat accessibility issues as bugs. If you encounter something that doesn't work, please tell us; we will work to fix it.

How we build for accessibility

The site is built with the following accessibility principles in mind:

  • Semantic HTML. Pages use proper landmarks (header, nav, main, footer), one h1 per page, and a sensible heading hierarchy.
  • Keyboard navigation. Everything you can do with a mouse, you can do with the keyboard. Tab order matches visual order; menus, dropdowns, and the cart drawer are operable via Tab, Enter, Escape, and arrow keys.
  • Visible focus. Every interactive control shows a clear focus outline when reached by keyboard.
